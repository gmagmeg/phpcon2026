### 概要

PHP8.0 で導入された OPcache と JIT は、現在も改善が続いています。
その一方で両方ともデフォルト設定のまま運用しているケースは多々あり、それぞれの特性を活かせていると言い切れる現場も多くないはずです。
例えば計算量について、 in_array より isset のほうが速い、といった断片的な知識は広く知られていますが
なぜ isset のほうが速いのか、そこに OPcache や JIT は効くのか、といった踏み込んだ理解はあまり共有されていません。
本セッションでは、こうしたなんとなく知っていることを、どこに、どう効くのかを測って確かめたに置き換えることが目的としています。

### 話すこと

PHP 標準関数や言語機能を計算量で選ぶ視点（in_array / isset / array_unique など）
OPcache・JIT の設定がパフォーマンスに与える影響
計測と実際の計測結果

### 話さないこと

OPcache、JITの踏み込んだ内部実装の話
JIT コンパイラ自体の理論的背景
