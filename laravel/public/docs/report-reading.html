PHPBench aggregate レポートの読み方

vendor/bin/phpbench run ... --report=aggregate の出力を読む

サンプル出力

+-----------------+-------------------------+-----+------+-----+----------+----------+--------+
| benchmark       | subject                 | set | revs | its | mem_peak | mode     | rstdev |
+-----------------+-------------------------+-----+------+-----+----------+----------+--------+
| CollectionBench | benchCollectionPipeline |     | 2000 | 5   | 6.909mb  | 99.151μs | ±0.53% |
| CollectionBench | benchRawArray           |     | 2000 | 5   | 6.909mb  | 6.304μs  | ±1.59% |
| CollectionBench | benchArrayFunctions     |     | 2000 | 5   | 6.909mb  | 49.285μs | ±0.32% |
+-----------------+-------------------------+-----+------+-----+----------+----------+--------+

各列の意味

列意味
benchmarkベンチクラス名（CollectionBench = benchmarks/CollectionBench.php）
subject計測対象メソッド名（bench〜 の各メソッド）
setパラメータセット名。@ParamProviders でデータを差し替えた時だけ入る。未使用なら空
revs1 iteration 内でメソッドを繰り返す回数（例 2000）。1回が短すぎて測れないのを均すため
itsiteration 数（例 5）。各 iteration は別プロセスで実行。この5個のばらつきから統計を出す
mem_peakプロセスのメモリ使用ピーク（≒アプリ起動分が主。メソッド間の差は小さい）
mode代表値（速さの本命）。最頻値（KDE：カーネル密度推定によるピーク）であり、単純平均・中央値ではない。外れ値に強く、実用上は中央値に近い。1回あたりの実行時間で、小さいほど速い。平均も見たいなら mean 列を足す
rstdev相対標準偏差。ばらつきの割合。小さいほど計測が安定＝信頼できる

数値の読み取り例

subjectmoderstdev素のforeach比
benchRawArray（素の foreach）6.304μs±1.59%1.0倍（最速・基準）
benchArrayFunctions（array_*）49.285μs±0.32%約7.8倍
benchCollectionPipeline（Collection）99.151μs±0.53%約15.7倍

単位 μs はマイクロ秒（1μs = 0.001ms）。rstdev が全て ±2%未満なので、この差は誤差ではなく有意と判断できる。

読む順番のコツ

  1. まず rstdev を見る — 大きい（目安 ±5〜10%超）と mode が信用できない。its や revs を増やして測り直す。
  2. rstdev が小さいことを確認してから mode を比較 — 安定した状態でのみ速さを比べる。
  3. mem_peak はメモリも見たい時に。各行ほぼ同じなら差は実行時間に出ている。

列を増やしたい場合

平均・最小・標準偏差なども出せる。例:

# mean(平均) / min(最小) / stdev(標準偏差) などを追加
docker compose exec app vendor/bin/phpbench run benchmarks/CollectionBench.php \
  --report='{"extends":"aggregate","cols":["subject","mode","mean","min","stdev","rstdev"]}'

総実行時間を見たい場合

標準の mode/mean は1回あたりに正規化された値で、総時間は出ない。総時間（revs × its 回分の合計＝実際に計測に費やした時間）を出すには、カスタム列で sum(result_time_net) を足す。

docker compose exec app vendor/bin/phpbench run benchmarks/BcmathBench.php \
  --report='{"extends":"aggregate","cols":{
    "subject":"first(subject_name)",
    "revs":"first(variant_revs)",
    "its":"count(result_time_avg)",
    "per_call":"mode(result_time_avg) as time",
    "total":"sum(result_time_net) as time"
  }}'
subjectrevsitsper_call（1回あたり）total（総時間）
benchNativeFloat20051.255μs1.264ms
benchBcmath2005264.683μs265.560ms

result_time_net = 各 iteration の生の合計時間（= revs 回分）。それを sum() で全 iteration 合算すると総時間になる。検算: per_call × revs × its ≈ total（1.255μs × 200 × 5 ≈ 1.26ms）。なおプロセス起動やブートまで含めた壁時計が欲しいなら time docker compose exec app vendor/bin/phpbench run ... を使う。
