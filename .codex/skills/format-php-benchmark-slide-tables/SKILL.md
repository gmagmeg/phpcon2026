# PHPベンチマーク表を整形する

GitHub Actionsのx86_64環境で取得したPHPBench結果を、比較条件と換算方法が伝わる表へ反映する。ローカルでは計測しない。

## 1. 計測条件を確認する

1. 対象のベンチマークPHPファイルを読む。
2. 1回のベンチマーク関数内で対象処理が実行される回数 `N` を特定する。`Revs` はPHPBenchが関数を呼ぶ回数であり、`N` と混同しない。
3. 各対象メソッドに対応する基準メソッドを特定する。
4. JIT OFF、JIT function、JIT tracingについて、対象と基準の未丸め値をそろえる。

`N` や対応する基準メソッドをコードから確定できない場合は、計算せずユーザーへ確認する。

## 2. 基準差引後の値を計算する

JIT設定ごとに次を計算する。

```text
基準差引後の総時間 = 対象メソッドの時間 - 同じJIT設定の基準メソッドの時間
```

- 丸める前の値で差し引く。
- 異なるJIT設定の基準値を流用しない。
- 基準値がない場合は「基準差引後」と表示しない。基準を追加するか、生値として掲載するかをユーザーへ確認する。
- 差し引きは分岐などの純粋な単体コストではなく、共通処理を除いた推定値として扱う。

## 3. １回あたりへ換算する

基準差引後の総時間が `μs` の場合、各JIT設定について次を計算する。

```text
1回あたり(ns) = 基準差引後の総時間(μs) × 1000 ÷ N
```

3つのJIT設定の換算値から最大値と最小値を選び、最後にスライドの表示精度へ丸める。途中値を先に丸めない。

表示形式は次に固定する。

```text
最大値 / 最小値 ns
```

例: `6.1 / 3.3 ns`

## 4. 表を組み替える

次の5列を基本形とする。

| 判定 | JIT OFF | function | tracing | １回あたり |
|---|---:|---:|---:|---:|

- JIT 3列には基準差引後の総時間を表示し、単位を各値へ付ける。例: `61.3μs`。
- 最後の列には `最大値 / 最小値 ns` だけを表示する。
- 単位を赤字や太字で強調しない。
- 既存の青い `key` 強調は、JITの短縮効果を示す用途なら維持する。
- 表の直前に `{N}回分（基準処理差引後の推定値）` と明記する。`N = 10000` なら `1万回分` とする。
- 表の直後に `１回あたり：最大値 / 最小値` と明記する。
- 行ごとに `N` が異なる場合は共通の回数を掲げない。回数ごとに表を分けるか、行名へ回数を添える。

既存の表デザインを優先し、5列で折り返す場合だけ専用クラスを追加する。既存の5列表用CSSを再利用・汎用化し、同じCSSを重複して作らない。

## 5. 反映後に検証する

1. `index.html` と必要最小限のCSSだけを編集する。
2. ブラウザで対象スライドを再読み込みする。
3. 見出し、5列のヘッダー、各換算値、注記を確認する。
4. セルの折り返し、文字切れ、表や注記のスライド外へのはみ出しがないことをDOMとスクリーンショットで確認する。
5. 本文相当の文字サイズは16px未満にしない。収まらない場合は、先に余白・文言・列幅を調整する。
6. `git diff --check` を実行し、対象外の既存変更を上書きしていないことを確認する。

## 6. 報告する

変更したスライドとファイルを示し、次を簡潔に報告する。

- 基準差引後の総時間を掲載したこと
- 1回あたりを最大値／最小値で換算したこと
- 使用した `N`
- レイアウト検証結果
